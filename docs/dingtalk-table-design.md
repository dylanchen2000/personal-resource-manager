# 钉钉多维表格设计 - PRM数据底座

## 设计原则

1. **钉钉只存安全信息** -- 即使被同事看到也不尴尬
2. **敏感数据在本地** -- 人情账本、策略标记、私密备注不上云
3. **字段命名规范** -- 中文字段名，方便直接在钉钉中查看

## 表1: 联系人（主表）

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 姓名 | text | Y | 联系人姓名 |
| 手机 | phone | N | 手机号码 |
| 微信 | text | N | 微信号 |
| 公司 | text | N | 所在公司/组织 |
| 职位 | text | N | 职位/角色 |
| 所属圈子 | multiSelect | N | 可多选：戈友圈/沙友圈/校友圈/体育圈/同好圈/商业圈/家人亲友 |
| 最近联系日期 | date | N | 最后一次互动的日期 |
| 备注 | text | N | 公开备注（安全信息） |
| 创建时间 | createdTime | auto | 系统自动 |
| 更新时间 | modifiedTime | auto | 系统自动 |

**不在钉钉中存放的字段（仅本地）：**
- 人情标记（有恩于我/有恩于他）
- 人情详情
- 关系策略（加密/保持/淡出）
- 私密备注

## 表2: 圈子

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 圈子名称 | text | Y | 如"戈友圈""校友圈" |
| 描述 | text | N | 圈子简介 |
| 颜色标识 | singleSelect | N | 用于前端展示颜色 |
| 成员数 | formula | auto | 关联联系人表统计 |

## 表3: 互动记录

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 联系人 | link | Y | 关联联系人表 |
| 互动日期 | date | Y | 互动发生日期 |
| 互动类型 | singleSelect | Y | 见面/电话/微信/饭局/帮忙/活动/送礼/其他 |
| 内容 | text | N | 互动内容描述 |
| 创建时间 | createdTime | auto | 系统自动 |

## 默认圈子选项

| 圈子 | 颜色 | 说明 |
|------|------|------|
| 戈友圈 | 黄色 | 戈壁挑战赛结识的朋友 |
| 沙友圈 | 红色 | 沙漠挑战赛伙伴 |
| 校友圈 | 蓝色 | 同窗校友 |
| 体育圈 | 绿色 | 运动相关朋友 |
| 同好圈 | 紫色 | 共同爱好者 |
| 商业圈 | 靛蓝 | 商业合作伙伴 |
| 家人亲友 | 粉色 | 家人和亲戚 |

## 本地隐私层数据模型

以下数据只存在本地浏览器 IndexedDB 中：

```
contacts: {
  id, name, phone, wechat, company, title,
  circles: string[],     // 所属圈子名
  favor: '有恩于我' | '有恩于他' | null,
  favorDetail: string,   // 恩情具体描述
  strategy: '加密' | '保持' | '淡出' | null,
  notes: string,         // 公开备注（可同步到钉钉）
  privateNotes: string,  // 私密备注（永不同步）
  lastContact: timestamp,
  createdAt: timestamp,
  updatedAt: timestamp
}

interactions: {
  id, contactId,
  type: '见面'|'电话'|'微信'|'饭局'|'帮忙'|'活动'|'送礼'|'其他',
  content: string,
  date: timestamp
}
```

## 同步策略（后续实现）

```
钉钉多维表 ←→ 本地PWA

同步方向：双向
同步范围：仅联系人基础信息 + 互动记录
不同步：人情标记、策略、私密备注
冲突策略：以最后更新时间为准
```
