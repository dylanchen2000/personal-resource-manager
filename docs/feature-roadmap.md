## 2. 功能详细规格说明

---

### 功能 1：圈层管理（Circle Management）

**用户故事**

作为一个用户，我希望把我的联系人按照关系类型和战略价值分组，并为每个圈层设定不同的维护规则，这样我就不会对所有人采用同样的互动策略，而是能分配精力到最重要的关系上。

**核心功能**

- 预设圈层模板（核心圈/战略伙伴/导师/潜力联系人/泛熟人）
- 用户可自定义圈层名称、颜色、图标
- 每个圈层独立配置：
  - 建议联系频率（每周/每月/每季/每年）
  - 互动深度期望（寒暄/实质性交流/战略协作）
  - 优先级权重（影响AI推荐排序）
- 联系人可同时归属多个圈层
- 圈层健康度聚合视图（当前圈层整体维护状态）

**UI/UX考量**

- 主界面采用同心圆可视化，最内圈是核心联系人，向外递减
- 拖拽操作移动联系人圈层归属
- 圈层配置页使用滑块设定频率，避免下拉菜单带来的认知负荷
- 颜色编码贯穿全应用，联系人卡片边框颜色对应所属最高优先级圈层

**技术复杂度：低**

数据模型层面是标准的多对多关系（联系人 — 圈层），核心逻辑是基于规则的分类。主要工作量在UI的同心圆渲染（可用D3.js或Canvas实现）。

### 功能 2：关系健康度评分（Relationship Scoring）

**用户故事**

作为用户，我希望系统自动计算我与每个联系人的关系健康状态，这样我不需要凭感觉判断，而是有一个量化依据告诉我哪些关系在衰退，哪些正在增强。

**核心功能**

评分算法由四个维度构成，总分0-100：

| 维度 | 权重 | 计算逻辑 |
|------|------|----------|
| 互动频率分 | 35% | 实际联系次数 / 圈层设定频率目标，超出目标100分，0次联系趋近0分，采用对数衰减曲线 |
| 互动互惠分 | 25% | 用户主动发起次数 vs 对方主动次数的比率，双向均衡得高分 |
| 情感质量分 | 25% | 用户对每次互动打的情感标签（正向/中性/负向）加权平均，近期互动权重更高 |
| 目标对齐分 | 15% | 联系人在SWOT匹配中的关联度评分，静态更新 |

附加机制：
- 时间衰减：所有历史互动按指数衰减，6个月前的互动权重降为25%
- 趋势指示器：近30天评分变化方向（上升/稳定/下降）
- 圈层基准对比：与同圈层平均分对比，识别相对弱关系

**UI/UX考量**

- 联系人卡片显示评分进度环，颜色从红（0-40）到黄（41-70）到绿（71-100）
- 评分详情页展开四个维度雷达图
- 趋势折线图显示过去90天评分变化
- 避免"分数焦虑"的设计：使用"需要关注"等语言化标签，数字分数仅在展开详情时显示

**技术复杂度：中**

核心难点在于时间衰减的实时计算性能优化，建议采用预计算缓存 + 定时刷新策略。情感质量分在P0阶段由用户手动打标，P1阶段接入本地NLP模型对备注文本做情感分析。

### 功能 3：AI沟通建议（AI Communication Advisor）

**用户故事**

作为用户，当系统提示我该联系某人时，我不只是想收到一个"去联系他"的提醒，而是希望得到具体的话术建议和对话起点，这样我能立即行动，而不是因为不知道说什么而拖延。

**核心功能**

触发条件引擎：

- 关系健康度低于阈值触发
- 联系频率超期触发
- 用户日历中有相关事件（开会/活动）时触发
- 联系人里程碑事件（生日/工作周年/新职位）触发

建议生成逻辑（分层）：

**第一层 — 触发上下文组装**
系统从以下字段提取相关信息：联系人最近互动摘要、对方当前职位与动态、上次对话话题标签、用户当前目标与需求、季节/节假日上下文。

**第二层 — 提示词模板 + LLM生成**
将上下文注入结构化提示词，调用本地或云端LLM生成3条备选建议，覆盖不同语气（正式/轻松/专业）。

**第三层 — 用户反馈学习**
用户选择/编辑/忽略建议后，系统记录偏好，逐步调整个人化风格。

建议输出示例：
```
联系人：张明（战略圈，产品方向）
上次联系：47天前，话题：新产品讨论
触发原因：关系健康度下降至52分

建议 A（轻松）：
"张总，上次聊到你们新品发布，最近推进得怎么样了？
我这边刚看完一篇关于B端SaaS定价策略的报告，感觉跟你们场景很像，要不要发给你？"

建议 B（专业）：
"张明，距我们上次深聊已近两个月。
最近我在关注[具体行业趋势]，想到与你此前提到的战略方向高度相关，
希望找个时间交流15分钟，同步一下各自的最新判断。"
```

**UI/UX考量**

- 建议以卡片形式呈现，一键复制到剪贴板，或直接跳转至微信/邮件应用
- 建议卡片提供"太正式/太随意/话题不对"的快速反馈按钮
- 首次使用引导：展示AI建议的生成逻辑，建立用户信任
- 隐私提示：明确告知哪些数据被用于生成建议

**技术复杂度：高**

关键技术决策：P1阶段优先使用OpenAI API或本地Ollama（支持隐私模式），上下文拼装逻辑需精心设计以控制Token成本。个人化学习模块可在P2阶段通过fine-tuning或few-shot examples实现。

### 功能 4：策略引擎（Strategy Engine）

**用户故事**

作为用户，我希望能对每个联系人设定明确的战略意图——是要主动深化、维持现状、还是有序淡出，系统根据我的意图自动生成对应的行动计划，而不是对所有人都推送"去联系"的通知。

**核心功能**

三种战略标记：

| 标记 | 中文标签 | 含义 | 系统行为 |
|------|----------|------|----------|
| Strengthen | 加密 | 主动投资这段关系 | 提高联系频率建议，生成深度互动计划，优先推送AI建议 |
| Maintain | 保持 | 维持当前关系温度 | 按设定频率提醒，保持现有互动深度 |
| Phase Out | 淡出 | 有序降低互动密度 | 停止主动提醒，记录最后一次礼貌互动，归档而非删除 |

行动计划生成：

对于"加密"联系人，系统生成结构化的30/60/90天计划：
- 第1周：重建联系点（发送一条有价值信息）
- 第2-4周：创造实质性互动（会面/深度通话）
- 第2个月：寻找互助机会（项目合作/资源对接/相互引荐）
- 第3个月：固化关系（建立定期沟通机制）

对于"淡出"联系人，系统提供有尊严的退出建议：
- 最后一次互动的推荐话题（避免突然断联的尴尬）
- 关系归档（数据保留，不计入健康度评分）
- 必要时重新激活路径

**UI/UX考量**

- 策略标记通过联系人详情页的三个图标选择，不使用下拉菜单
- "淡出"操作需要一次确认，附带说明（"此联系人将从每日建议中移除，数据保留"）
- 行动计划以时间线卡片展示，可逐项勾选完成
- 仪表板提供战略分布视图：当前加密/保持/淡出的联系人数量与健康度汇总

**技术复杂度：中**

核心是规则引擎 + 提示词模板，行动计划生成在P1阶段可采用模板化逻辑，P2阶段升级为LLM动态生成。

### 功能 5：个人SWOT集成与匹配引擎（Personal SWOT Integration）

**用户故事**

作为用户，我希望将自己当前的优势、劣势、机会和威胁输入系统，让系统告诉我哪些联系人可以帮助我弥补短板或放大优势，从而让我的人际网络真正服务于我的战略目标。

**核心功能**

SWOT录入模块：
- 引导式输入问卷（非空白表单），提供维度提示
- 每个SWOT条目可关联标签（技能/资源/行业/人脉）
- 支持季度更新，历史版本对比

联系人能力标签体系：
- 联系人档案中增加"能力/资源/影响力"标签字段
- 标签来源：用户手动标注 + AI从互动记录中提取建议

匹配算法逻辑：

```
匹配分 = Σ(SWOT条目权重 × 联系人相关标签匹配度)

劣势匹配：寻找能补充用户不足的联系人（权重最高）
优势放大：寻找能与用户优势产生协同的联系人
机会把握：寻找在目标机会领域有资源的联系人
威胁对冲：寻找能提供信息或保护的联系人
```

输出：
- "战略价值最高的5个联系人"推荐列表（附原因说明）
- "被忽视的高价值联系人"预警（高匹配度但近期互动不足）
- "资源缺口"分析：哪些SWOT条目在网络中找不到对应资源

**UI/UX考量**

- SWOT可视化为四象限卡片，每个条目显示"已覆盖/部分覆盖/未覆盖"状态
- 匹配结果以关系图谱形式展示：用户在中心，联系人节点按匹配度远近分布
- 引导用户建立"资源地图"思维模型，而非功能性标签列表

**技术复杂度：中**

标签匹配是基础向量相似度问题，可用TF-IDF或小型Embedding模型实现。关键挑战是引导用户持续维护SWOT内容的产品设计，建议通过季度回顾流程强化。

### 功能 6：隐私分级（Privacy Tiers）

**用户故事**

作为用户，我的联系人档案中包含非常敏感的私人判断（例如"这个人不可信任"或"竞争对手"），我希望这些信息有最高级别的保护，与普通联系信息严格分离。

**核心功能**

三级隐私模型：

| 级别 | 内容类型 | 存储方式 | AI访问权限 |
|------|----------|----------|------------|
| 公开层 | 基本联系信息、公开职业信息 | 普通数据库 | 完全可用 |
| 私密层 | 互动记录、关系评分、备注 | 本地加密存储 | 本地模型可用，云端不可用 |
| 核心层 | 战略标记（加密/淡出）、个人判断、敏感标签 | 端对端加密，仅设备本地 | 仅离线AI可用 |

功能配置：
- 用户可以为每个联系人字段单独设置隐私级别
- 隐私模式：开启后所有AI建议仅在本地处理，不上传任何数据
- 数据导出时：自动过滤核心层内容，防止意外泄露
- 审计日志：记录哪些操作访问了私密/核心层数据

**UI/UX考量**

- 隐私级别以锁图标直观区分（开锁/单锁/双锁）
- 首次设置引导：明确解释三层数据的使用方式
- 编辑核心层内容时显示"仅在此设备可见"提示
- 避免隐私设置过于复杂，提供"一键锁定所有敏感内容"快捷操作

**技术复杂度：高**

核心挑战是端对端加密密钥管理（设备级密钥 vs 密码派生密钥），以及本地LLM部署（支持隐私模式的离线AI建议）。推荐技术栈：SQLCipher（本地加密数据库）+ Ollama（本地LLM）。

### 功能 7：自动化引擎（Automation Engine）

**用户故事**

作为用户，我希望系统能代替我执行低难度但高频的关系维护动作，如生日祝福、定期问候，让我把精力集中在真正需要我亲自投入的高价值互动上。

**核心功能**

自动化规则构建器：

```
触发条件（When）：
- 日期事件：生日、工作周年、节假日
- 关系事件：健康度低于X、超过X天未联系
- 外部事件：LinkedIn职位变更（P2），日历事件

执行动作（Then）：
- 发送提醒到用户（P0/P1）
- 起草消息草稿供用户审核（P1）
- 自动发送（仅低风险场景，用户明确授权）（P2）

过滤条件（But only if）：
- 联系人策略标记为"保持"或"加密"
- 距上次联系超过X天
- 时间在工作日X时至X时之间
```

集成渠道（优先级排序）：
1. 应用内提醒 + 草稿（P0）
2. 系统通知（P0）
3. 邮件发送（P1）
4. 微信（通过企业微信API或半自动化剪贴板方式）（P1-P2）
5. LinkedIn（P2）

**UI/UX考量**

- 自动化规则使用If-Then-But可视化构建界面，非代码配置
- 规则列表显示"下一次触发时间"，增强透明度
- 自动发送前提供24小时取消窗口（不是即时执行）
- 发送历史日志独立页面，用户随时可审计

**技术复杂度：中至高（随集成渠道增加而提升）**

P0阶段复杂度低（本地通知 + 草稿），微信集成是主要技术难点（需评估企业微信API可行性与个人账号方案的合规性）。

### 技术选型建议

| 层次 | 推荐方案 | 备选方案 |
|------|----------|----------|

### Sprint计划（每Sprint 2周）

#### 第一阶段：MVP基础（Month 1-2，Sprint 1-4）

**Sprint 1：数据基础层**

目标：能录入联系人，能创建圈层

- 联系人数据模型设计与数据库Schema
- 基础CRUD：创建/编辑/删除/搜索联系人
- 圈层创建与联系人归属
- 本地SQLite数据库初始化
- 基础导航框架（Tab + Stack导航）

交付物：可以录入联系人和圈层的功能原型

**Sprint 2：互动记录与提醒**

目标：能记录互动，能收到提醒

- 互动记录创建（类型/备注/情感标签）
- 互动历史时间线视图
- 本地推送通知系统
- 生日提醒（基于联系人生日字段）
- 联系频率到期提醒逻辑

交付物：完整的互动日志功能 + 基础提醒系统

**Sprint 3：关系健康度（静态版）+ 个人资料**

目标：能看到关系评分，能设置个人SWOT

- 关系健康度评分（频率维度）实现
- 联系人卡片评分可视化
- 用户个人资料页（SWOT录入）
- 圈层配置页（频率/优先级设置）

交付物：带评分的联系人列表 + SWOT输入模块

**Sprint 4：MVP打磨与测试**

目标：MVP可供内测用户使用

- 数据导入（通讯录导入、CSV导入）
- 搜索与过滤功能完善
- 同心圆可视化圈层视图
- 性能优化（列表渲染、查询缓存）
- 内测发布（TestFlight / Android APK）

关键决策点：内测反馈收集，确认P1功能优先级排序

---

#### 第二阶段：核心差异化（Month 3-4，Sprint 5-8）

**Sprint 5：关系健康度（多维版）**

- 互动互惠分计算逻辑
- 情感质量分（手动标签）
- 时间衰减算法实现
- 趋势指示器（30天变化）
- 四维雷达图UI

**Sprint 6：AI沟通建议（基础版）**

- OpenAI API集成

### 核心差异化叙事：环境优化（Environment Optimization）

现有个人CRM工具的思维模型是"更好地管理关系"，本产品的思维模型是"主动设计你的人际环境"。

这一叙事差异体现在以下三个层面：

**第一层：主动塑造 vs 被动记录**

传统CRM工具（包括Dex、Monica）的核心价值主张是"不要忘记联系某人"，本质是提醒工具。本产品的价值主张是"识别出哪些人应该进入你的生活，哪些人应该优雅退出"。这是从被动记录到主动规划的范式转变。

**第二层：自我战略对齐**

SWOT集成功能将人际网络与个人目标显式连接。用户不只是管理联系人，而是在回答"我的人际环境是否支撑我的战略目标？"这一问题。这一功能在现有任何个人CRM中均无对应产品。

**第三层：隐私优先的AI**

在数据隐私意识日益增强的市场环境中，本地AI模式（完全离线处理）是面向高敏感度用户（高管、律师、医生、创业者）的核心卖点。这使产品能进入其他数据上云的CRM工具无法进入的细分市场。

### 目标用户画像与切入策略

**核心用户：主动经营人生的"资源意识"人群**

- 创业者（初创/小企业主）：关系是核心生产资料
- 高管/职场发展人群：晋升路径上的关键关系管理
- 自由职业者：客户维护与口碑扩展完全依赖人脉
- 个人成长爱好者：对自我优化方法论高度认可

**切入策略：内容营销 + 社群种子**

通过"你的人际网络是你的第二大脑"的内容叙事，在即刻、小红书、播客平台建立认知，吸引对个人成长方法论有高度认同感的种子用户群体，形成口碑传播飞轮。
