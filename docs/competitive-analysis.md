## 一、Dex Personal CRM

**定位**: 聚合LinkedIn、Gmail、Calendar等所有联系人来源，以"人"为中心替代分散工具。

### 核心功能（可借鉴）

**1. 自动职位变更同步**
- 当联系人在LinkedIn更换工作时，Dex自动更新联系人档案
- 实现方式：LinkedIn浏览器扩展 + 后台定时抓取
- PRM借鉴：微信变更签名/职位时触发通知（用钉钉Webhook监听）

**2. 浏览器扩展"原地操作"**
- 在LinkedIn/Facebook/Gmail页面直接添加备注、设置提醒，无需切换应用
- 核心UX：不打断用户工作流，在信息发生的地方记录信息
- PRM借鉴：微信端直接触发快速备注（通过小程序或钉钉插件）

**3. 关系上下文记忆**
- 存储字段：认识经过、教育背景、工作史、重要日期、家庭信息、礼物想法、事件历史
- "上次停在哪里"功能：再次联系时快速回顾
- PRM借鉴：联系人"快速上下文卡片"，每次打开先看最近动态

**4. 多平台统一收件箱**
- 集成Gmail、Outlook、iMessage、WhatsApp、Instagram、X
- 在联系人档案页显示所有渠道的对话历史时间线
- PRM借鉴：将微信互动记录手动添加到统一时间线（输入框快速记录）

**5. 隐私订阅模式**
- 无广告、用户数据不出售，订阅付费支撑
- 提供数据导出和删除功能
- 30,000+ 用户规模

### UI/UX 亮点
- 联系人主页：顶部显示"最近交流"badge（距上次联系N天）
- 分组功能：标签 vs 动态分组（按公司、按互动频率）
- 提醒页面：今日待跟进列表，不是日历视图而是任务流

### 数据模型
```
Contact:
  - how_we_met: text
  - gift_ideas: list
  - important_dates: [{label, date, recurring: bool}]
  - family_members: [{name, relationship}]
  - last_contact: datetime
  - follow_up_reminder: {date, note}
Groups: many-to-many
Integrations: {platform, external_id, last_synced}
```

## 二、Monica Personal CRM（开源，21k stars）

**定位**: 开源PRM，帮助管理家人和朋友关系，明确说"不是社交网络"，支持自建。

### 核心功能（最值得抄）

**1. 礼物管理模块**
- 记录已送礼物、已收礼物（含日期、价格、场合）
- 生日/节日前自动提醒"该准备礼物了"
- 礼物想法列表（wishlist for others）
- PRM借鉴：人情账本的"礼物往来"维度，记录送礼/收礼，计算礼尚往来平衡

**2. 债务追踪**
- 记录欠人钱 / 别人欠我钱
- 支持标记"已还清"
- PRM借鉴：直接对应"人情账本"的核心功能，比单纯记账更有人情味

**3. 互动记录（Activity Log）**
- 记录每次活动：日期、地点、参与者（可多人）、活动类型（自定义）、备注
- 可记录"花了多少时间"
- 支持自定义活动类型（吃饭/打球/聊天/帮忙）
- PRM借鉴：互动日志是人情关系的核心记录，每次见面/帮忙都记录

**4. 关系图谱（Contact Relationships）**
- 联系人之间可建立关系（A的妻子是B、A介绍认识C、A和B同属于某圈子）
- PRM借鉴："圈子"概念正好对应此数据模型，人 -> 圈子 -> 人

**5. 多保险库（Multiple Vaults）**
- 可创建多个独立空间，互相隔离
- 场景：工作人脉 vs 私人朋友 vs 家庭
- PRM借鉴：对应"圈层"概念，戈友、沙友、家人可以分库管理

**6. 可定制字段类型和区块**
- 可以自定义"联系方式类型"（微信/抖音/小红书）
- 可以定义哪些字段区块在联系人页面显示/隐藏
- PRM借鉴：用户可以按自己需求增减字段，不强迫填所有信息

**7. 宠物信息**
- 记录联系人的宠物（名字、品种、生日）
- 看似奇怪但用户极爱：聊天时能记住对方的狗叫什么
- PRM借鉴："关心细节"模块：孩子、宠物、爱好、特殊偏好

### 数据模型（最完整的参考）
```
Contact:
  - name, nickname, gender(自定义), pronouns
  - birthdate, deceased_date
  - how_we_met: text
  - notes: [{body, is_private}]
  - tags/labels: many-to-many
  - relationships: [{type, contact_id}]  # "是...的"
  - addresses: [{type, full_address}]
  - contact_fields: [{type, data}]  # 自定义联系方式
  - activities: [Activity]
  - reminders: [Reminder]
  - gifts: [{name, status: idea/given/received, date, price}]
  - debts: [{amount, reason, in_or_out, resolved}]
  - documents: [File]
  - pets: [{name, type, birthdate}]

Activity:
  - date, summary, description
  - participants: [Contact]
  - activity_type: {自定义}
  - duration_minutes

Reminder:
  - initial_date, frequency: {once/weekly/monthly/yearly}
  - title, description
  - notification_sent: bool

Journal:
  - date, title, post (日记，独立于联系人)
```

### 用户爱它的原因
- 自建 = 数据完全私有，这是最大卖点
- 礼物 + 债务 + 宠物 = 有"人情味"的CRM，不是冷冰冰的联系人管理
- 简洁但完整，不过度设计

## 三、Clay (现在叫 Mesh / me.sh)

**定位**: AI驱动的人脉情报平台，自动聚合所有来源数据并主动推送再连接建议。

### 核心功能（最有创意）

**1. Filter Feed（多维度关系动态流）**
这是Clay/Mesh最独特的功能，类似"人脉版朋友圈"但可筛选：
- Birthdays：生日提醒
- Events：联系人参加的活动
- News：联系人相关新闻
- Social changes：换工作、搬家、结婚等人生事件
- Posts：联系人的社交媒体内容
- Reconnect prompts：AI建议"该联系XX了"
- Reminders：手动设置的提醒
- Upcoming：即将到来的互动

用户可以选择显示哪些类别，Feed是按时间排列的关系动态流，不是静态联系人列表。

**PRM借鉴**: 首页不是"联系人列表"，而是"今日关系动态流"：
- 今天生日的人
- 超过N天未联系的人（变色预警）
- 手动记录的待跟进事项
- 圈子活动提醒

**2. 互动历史统计**
"告诉你和某人互动了多少次、通过什么渠道、在什么时间段"
- 自动统计跨渠道互动频次
- 可视化：过去30天/90天/1年的接触热力图
- PRM借鉴：联系人详情页显示"关系热力图"，帮助判断是否需要维系

**3. 再连接提示（Reconnect Prompts）**
- AI分析互动间隔，主动建议"你已经X个月没联系Y了，建议重新联系"
- 用户可设置每个联系人的"保持联系频率"（每月/每季/每年）
- 提示附带"上次聊了什么"的简要回顾
- PRM借鉴：这正是"关系策略"的自动化实现

**4. 团队共享网络（Who knows whom）**
- 显示团队中谁认识某个外部联系人
- 方便借助内部人脉做热情引荐
- PRM借鉴：在"圈子"维度可以看"谁和谁共同认识"，用于引荐或活动组织

**5. "世界级"极简界面**
- 用户描述：极简、用户友好
- 联系人卡片：关键信息"一眼可见"，不需要滚动
- 颜色编码：不同接触频率用颜色区分关系健康度

### UI模式亮点
- "动态流"首页，而非静态列表首页
- 每个联系人卡片顶部有"关系健康度"色条（绿/黄/红）
- 空状态引导：当Feed空时，提示"添加联系人开始追踪"

## 四、Folk CRM

**定位**: 轻量CRM，面向销售和个人关系，AI助手嵌入式工作流。

### 核心功能（AI方向最强）

**1. AI Recap 总结**
- 自动生成关系摘要："你和张三的互动历史摘要：去年3月通过LinkedIn认识，讨论过A项目，上次通话在X月，对方关注Y方向..."
- 在每个联系人页面顶部显示AI生成的关系简介
- PRM借鉴：联系人页面顶部"AI生成的一句话关系描述"，降低回忆成本

**2. Follow-up 时机检测**
- 分析邮件/WhatsApp内容，自动识别"这封邮件需要回复但还没回"
- 把遗漏跟进转化为待办任务
- PRM借鉴：结合微信消息，检测"最后一条是对方发来的，你还没回，已超N天"

**3. Magic Fields（AI一键丰富）**
- 输入公司名，AI自动填充：行业、规模、网站、职位
- 输入姓名，AI检索公开信息补充背景
- PRM借鉴："一键AI扩写"：输入姓名+公司，AI搜索背景资料生成简要介绍

**4. folkX Chrome扩展（LinkedIn一键导入）**
- 在LinkedIn页面，一键将该联系人导入CRM
- 自动抓取：姓名、职位、公司、头像
- PRM借鉴：微信名片一键导入（扫描对话截图或手动粘贴链接）

**5. 自定义管道视图（Pipeline）**
- 可以把联系人放入"关系进展阶段"：初识 -> 建立信任 -> 深度合作 -> 互为资源
- PRM借鉴："关系策略"的可视化看板：strengthen（深化）/ maintain（保持）/ phase out（淡出）

### 数据模型特点
- Groups / Lists：同一联系人可属于多个列表
- Custom Fields：每个列表可以有自定义字段（不同场景字段不同）
- Magic Fields：AI自动填充的字段有标记，区别于用户手动输入

## 五、UpHabit

**定位**: 习惯化关系维护，强调"规律接触"而非一次性管理。

### 核心功能（提醒系统最佳实践）

**1. 灵活提醒间隔（Preset + Custom Intervals）**
- 预设选项：每周/每两周/每月/每季度/每半年/每年
- 自定义：每N天
- 每个联系人单独设置频率（不是全局统一）
- PRM借鉴：关系策略与提醒频率绑定：
  - Strengthen（深化）= 每2周
  - Maintain（保持）= 每月
  - Phase out（淡出）= 不设提醒

**2. Bulk Snooze（批量推迟）**
这是UpHabit独有的天才功能：
- 当积累了太多待联系提醒时，可以"均匀分散"所有提醒
- 系统自动把堆积的提醒重新分配到未来N天，避免一天要联系20人的焦虑
- PRM借鉴：提醒溢出时，"智能分散"按钮一键重新排期

**3. 互动时间轴（Timeline）**
- 每次记录联系，都在联系人页面显示时间轴
- 时间轴上：何时联系、通过什么方式、写了什么备注
- PRM借鉴：联系人页面的互动历史时间线（仿微信聊天记录的形式）

**4. 联系人来源合并（Suggested Merge）**
- 自动识别重复联系人（同名/同手机号）
- 提示用户合并，保留所有备注和标签
- PRM借鉴：数据清洗功能，避免同一人在钉钉表中有多个条目

**5. 生日系统**
- 不只是生日当天提醒，还可以设置提前N天提醒（给你时间准备礼物/祝福语）
- PRM借鉴：生日前3天提醒"XX三天后生日，去年你送了[礼物]"

### UI模式
- 联系人列表按"下次需要联系"日期排序，而非字母顺序
- 即将到期的联系人显示倒计时徽章
- 主页是"今日需要联系"列表，而非全量联系人

## 六、Cloze（AI关系智能）

**定位**: 专为房产经纪人等重度关系行业，AI自动记录所有接触，主动识别商机。

### 核心功能

**1. 全自动多渠道记录**
- 电话、短信、邮件自动同步到对应联系人
- 无需手动记录，使用后台权限自动归档
- PRM借鉴：钉钉消息、通话记录自动关联到联系人（企业微信有类似功能）

**2. 每日AI议程**
- 每天早上推送"今日关系待办"：
  - 推荐联系X，因为上次联系已超N天
  - 提醒跟进Y，因为上次邮件没有回复
  - 重要日期：今天Z的周年纪念
- PRM借鉴：每日"关系日报"推送通知，而非被动等用户打开app

**3. 关系"再入局"检测**
- AI识别哪些老客户/老朋友"重新活跃了"（最近发了朋友圈/换了工作）
- 提示用户主动出击
- PRM借鉴：用钉钉表单的"最后联系"字段+公式自动标记"冷却中"关系

**4. 通讯内容语义分析**
- 分析邮件内容，自动提取：承诺事项、问题、情绪倾向
- 生成"对话摘要"
- PRM借鉴：AI分析最近一次互动记录，生成"上次聊了什么"的一行摘要

**5. 36% 销售提升主张**
- 基于用户数据的可量化结果
- PRM借鉴：给用户展示"关系健康度评分"，量化维系效果

## 七、Covve

**定位**: 移动端名片扫描 + 联系人情报，4.8分/33000评价。

### 核心功能

**1. 联系人新闻情报**
独特功能：在你联系某人**之前**，Covve推送与该联系人相关的新闻：
- 他的公司获得融资了
- 他所在行业有重大变化
- 适合作为"破冰话题"的内容

实现方式：根据联系人公司名/姓名，抓取公开新闻
PRM借鉴：联系人详情页底部"话题雷达"，显示该联系人最近的公开动态

**2. 关系健康度分析（Network Analytics）**
- 实时分析"你的网络是否健康"：
  - 多久没联系了（热/温/冷分层）
  - 各行业/渠道的分布
  - 最活跃的关系 vs 最被忽视的关系
- PRM借鉴：关系仪表盘：圆饼图显示各圈子分布，热力图显示联系密度

**3. 声音备注（Voice Capture）**
- 开完会后立即语音记录："刚和XX吃饭，他提到想做海外业务，下月再聊"
- AI转文字后关联到联系人
- PRM借鉴：快速语音备注是高频场景，特别是在没空打字的时候

**4. 名片扫描（60+ 语言）**
- 拍照即识别，支持中/英/日/韩等所有主流语言
- 识别率高于其他工具
- PRM借鉴：名片扫描 + 手动补充，是添加联系人最自然的起点

## 八、GitHub 开源项目分析

### 值得参考的开源项目

**Tilly**（62 stars）
- Progressive Web App + 离线优先
- 本地存储（IndexedDB）+ AI agent支持
- 设计理念与PRM高度一致

**FRM**（命令行工具）
- 基于CardDAV协议，复用手机通讯录
- AI驱动的互动日志
- 会议前自动准备联系人背景资料

**Personal-CRM（Django）**
- Docker部署，SQLite存储
- 极简数据模型，最小可用

## 九、用户高频需求（Reddit/论坛提炼）

基于各平台评论和评测综合分析：

### 用户最爱的功能
1. **自动提醒** - "帮我想起该联系谁"（所有工具共同高分项）
2. **简单快速记录** - "见面后10秒内能记下来"（输入摩擦必须极低）
3. **不打扰日常** - "不要每天要我主动打开app"（推送通知而非主动拉）
4. **私密性** - "我的关系数据不能上云"（特别是商业关系）
5. **跨平台同步** - "手机记的要在电脑上看到"

### 用户最抱怨的问题
1. **数据录入太重** - "要填的东西太多，几天就放弃了"
2. **提醒太多变噪声** - "开始时很有用，后来全部忽略"
3. **不知道该记什么** - "缺少引导，不知道哪些值得记"
4. **import/export 困难** - "数据被锁在里面，迁移成本高"
5. **价格** - "个人用途付订阅费有点重"

## 十、中国本土工具 & 微信生态

### 现有中文工具情况
- **销售type的CRM**（销售易、纷享销客）：面向B2B销售，过重
- **通讯录管理App**（如企业微信）：联系人管理，无"人情"维度
- **个人关系管理工具**：几乎空白，尚无针对"圈层管理+人情账本"的工具

### 微信生态特有机会
1. **微信名片直接导入**：分享名片截图 -> OCR识别 -> 自动创建联系人
2. **公众号动态追踪**：订阅联系人相关公众号，作为"动态情报"
3. **微信群人脉挖掘**：群成员 = 同圈子的人，标记"同属XX群"
4. **朋友圈动态作为互动记录**：看到朋友圈评论了 = 一次接触记录
5. **红包/转账作为"人情"**：微信支付记录可作为人情账本数据来源

### 钉钉生态整合（基于PRM技术架构）

## 十二、数据模型综合建议（适配钉钉多维表 + 本地存储分层）

### 云端层（钉钉多维表 - 基础档案）
```
联系人表:
  姓名, 英文名, 头像URL
  认识经过 (text)
  第一次见面日期
  圈子 (多选: 戈友/沙友/校友/同好/体育/工作/家人)

  提醒频率 (单选: 每周/每两周/每月/每季度/每年/不提醒)
  下次联系日期 (公式: 最后联系日期 + 频率)
  联系方式 (子表: 类型+值)
  标签 (多选)
  最后联系日期 (手动更新)

互动记录表:
  联系人 (关联)
  日期
  互动类型 (见面/电话/微信/帮忙/被帮/其他)
  一行摘要
  详细备注
  情绪标记 (愉快/普通/尴尬/需要跟进)

  圈子名称, 描述, 创建日期, 成员数
```

### 本地层（PWA IndexedDB - 隐私数据）
```
人情账本:
  联系人ID (关联钉钉行ID)

  下次聊天话题准备
```

## 十三、竞品差异化机会

基于以上研究，PRM的差异化机会在于：

**1. 人情味 > 效率工具**
所有英文工具都在强调"效率"、"不错过商机"。但中国用户的关系管理更关注"人情世故"，要有温度。

**2. 隐私分层设计**
所有竞品都是"全部上云"或"全部本地"。PRM的分层设计（基础信息上云、人情策略本地）是真正的创新。

**3. 圈层概念的原生支持**
没有任何工具有"圈层"（戈友/沙友）的概念。这是中国文化中特有的关系结构，应该成为PRM的核心概念。

**4. 人情账本的量化**
其他工具只记录礼物/债务，但没有"人情平衡"的概念。PRM可以显示"你对此人的人情净值"，帮助判断关系是否健康平衡。

**5. 移动端优先 + 微信深度整合**
所有英文工具都是Web优先。但中国用户的社交关系几乎全在微信，需要针对微信生态的特别设计。
