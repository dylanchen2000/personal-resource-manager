1 全球竞品分析

调研了全球范围内的个人CRM/关系管理工具，涵盖商业产品、开源项目和AI驱动工具。

商业产品

Dex (getdex.com)

Web+Mobile 30,000+ 用户

最接近的竞品。核心功能: LinkedIn/邮件集成、联系频率提醒、关系备注、生日提醒。

缺失: 无AI建议、无圈层战略标记、无SWOT匹配

Monica (monicahq.com)

开源 PHP/Laravel

开源个人CRM的标杆。记录互动、家庭关系、活动、礼物、笔记。GitHub 21k+ Stars。

缺失: 无AI功能、无自动化、无关系评分

Clay (clay.earth)

Web AI驱动

自动从邮件/日历/社交媒体同步联系人数据。有AI助手帮助筛选和推荐联系人。

缺失: 偏向B2B销售、无个人战略规划

Folk (folk.app)

Web 类Notion UI

轻量CRM，类似Notion的灵活界面。支持联系人管道管理、邮件序列。

缺失: 面向团队销售、无个人关系深度管理

UpHabit

iOS/Android 免费+付费

纯移动端关系管理。联系提醒、分组、互动记录。UI简洁，适合轻度用户。

缺失: 无AI、无战略规划、功能较浅

Notion + 模板

Web+Mobile DIY

很多用户用Notion数据库做个人CRM。灵活但需要自己搭建，无自动化。

缺失: 纯手动、无提醒、无AI、配置门槛高

AI驱动 / 创新工具

Claudia (GitHub开源)

开源 AI原生 终端CLI

基于Claude Code的终端AI助手。核心亮点: 关系健康度追踪、承诺追踪、模式识别、关系冷却预警。数据存储在Obsidian Vault中。197 GitHub Stars。

启发: AI原生+本地隐私优先是正确方向

PingCRM (GitHub开源)

开源 AI评分 多平台集成

AI驱动的开源个人CRM。关系评分(0-10)、Gmail/Telegram/Twitter同步、生命事件检测(职位变动/生日等)、Claude AI生成跟进建议。Python FastAPI + React。

启发: 关系评分+事件检测+AI建议的组合很有价值

Limitless AI (已被Meta收购)

已停运 可穿戴

AI可穿戴设备(Pendant)，自动录音、转录对话。虽非CRM，但代表了"自动记录人际互动"的方向。

启发: 自动化互动记录是未来趋势

grw (GitHub 842 Stars)

开源 已停维护

中文个人事务管理系统。JavaScript开发，包含个人网络管理功能。虽已停止维护，但说明中文市场有需求。

启发: 中文市场缺乏持续维护的个人关系管理工具

中国市场现状

中国市场目前没有专门的、成熟的个人关系管理App。现有替代方案:

微信生态

标签分组、备注功能，但无提醒、无AI、无关系评分。是最大的社交入口但非管理工具。

钉钉/飞书

企业CRM导向，不适合个人使用。钉钉多维表格可以做简单通讯录但无智能功能。

脉脉

偏职业社交网络，不是个人关系管理工具。数据归平台所有。

Notion/飞书多维表格 DIY

极客用户自建方案。门槛高、无自动化、需要持续维护。

核心结论: 三个独创差异化功能

1. 战略标记引擎

全球没有任何产品支持"加密/保持/淡出"的关系战略管理。这是从被动记录到主动设计的范式转变。

2. SWOT匹配

将个人优劣势与人脉资源对齐。回答"我的人际环境是否支撑我的战略目标"。

3. 三层隐私架构

公开层/私密层/核心层，敏感判断(如"此人不可信")仅设备本地加密存储。

3 产品核心概念: "环境设计"

不是"更好地管理关系"，而是"主动设计你的人际环境"。

▲

加密 (Strengthen)

主动投资的高价值关系。系统生成30/60/90天深化计划，优先推送AI建议。

▬

保持 (Maintain)

维持当前关系温度。按设定频率提醒，保持现有互动深度，不特别加力。

▼

淡出 (Phase Out)

有尊严地降低互动密度。停止主动提醒，建议最后一次礼貌互动话题，归档保留。

6 功能规划与优先级

分三期开发，MVP在2个月内可交付。

P0 MVP
第1-2月
  • 联系人管理 -- 录入/编辑/搜索/导入(通讯录/CSV)
  • 圈层管理(基础版) -- 创建圈层、分配联系人、设定联系频率
  • 互动记录(手动) -- 记录对话/见面/通话，打标签
  • 关系健康度(静态版) -- 基于最后联系时间的简单评分
  • 提醒系统(基础版) -- 生日提醒、联系频率到期提醒
  • 个人资料中心 -- 用户自身SWOT输入、目标设定
P1 差异化
第3-4月
  • AI沟通建议 -- 基于上下文生成对话起点、话题推荐、3种语气备选
  • 关系健康度(多维版) -- 频率/互惠/情感/目标四维评分 + 趋势分析
  • 策略引擎 -- 加密/保持/淡出标记 + 30/60/90天行动计划
  • SWOT匹配引擎 -- 联系人能力与用户需求的智能匹配
  • 隐私分级 -- 三层加密(公开/私密/核心)
P2 进阶
第5-6月
  • 全自动化引擎 -- If-Then-But规则构建、自动起草消息、邮件发送
  • 社交媒体集成 -- 微信(企业微信API)、LinkedIn动态监控
  • 高级分析仪表板 -- 网络图谱可视化、引荐路径推荐、资源缺口分析
  • 群体动态分析 -- 识别关系集群
  • 日历/邮件/CRM系统整合

AI沟通建议示例

联系人: 张明 (战略圈, 产品方向) 上次联系: 47天前, 话题: 新产品讨论 触发原因: 关系健康度下降至52分 建议 A (轻松): "张总，上次聊到你们新品发布，最近推进得怎么样了？ 我这边刚看完一篇关于B端SaaS定价策略的报告， 感觉跟你们场景很像，要不要发给你？" 建议 B (专业): "张明，距我们上次深聊已近两个月。最近我在关注 [行业趋势]，想到与你此前提到的战略方向高度相关， 希望找个时间交流15分钟，同步一下各自的最新判断。"

7 开发路线图

12个Sprint (每Sprint 2周)，6个月完成v1.0。

Sprint 1-2 | 数据基础 + 互动记录 (Month 1)

联系人CRUD、圈层创建、SQLite数据库、互动记录、基础提醒、导航框架

Sprint 3-4 | MVP打磨 + 内测 (Month 2)

关系评分(静态版)、SWOT输入、通讯录导入、同心圆可视化、TestFlight发布内测

决策点: 内测反馈收集，确认P1优先级

Sprint 5-6 | AI建议 + 多维评分 (Month 3)

多维关系评分(四维+趋势)、Claude API集成、上下文拼装、建议卡片UI、用户反馈学习

决策点: 评估本地LLM替代方案

Sprint 7-8 | 策略引擎 + 隐私分级 (Month 4)

加密/保持/淡出标记、行动计划生成、SWOT匹配、SQLCipher加密、三层隐私UI、App Store准备

Sprint 9-10 | 自动化 + 社交集成 (Month 5)

If-Then-But规则引擎、消息草稿自动化、邮件集成、微信半自动化方案评估

决策点: 微信集成技术方案最终确认

Sprint 11-12 | 高级分析 + v1.0发布 (Month 6)

网络图谱可视化、引荐路径推荐、资源缺口分析、性能优化、v1.0正式发布

9 下一步行动建议

具体可执行的下一步。

建议1: 先用MVP验证核心假设 (2周)

不急着写App，先用Notion/飞书多维表格搭一个简化版，自己用2周。验证:

- 圈层分类对你实际有帮助吗？分几层合适？

- 关系评分公式是否合理？

- 你真的会看AI建议并行动吗？

建议2: 技术选型二选一

方案A: 轻量Web App优先

Next.js全栈 + Supabase + PWA。2-3周出MVP，验证后再做Native。适合快速验证。

方案B: React Native原生优先

React Native + SQLite本地。体验更好，但开发周期长。适合确定要做产品。

建议3: 借鉴而非重造轮子

参考开源项目起步:

- PingCRM的关系评分+AI建议架构可以直接借鉴(Python FastAPI + React)

- Claudia的记忆系统和关系追踪模式值得研究

- Monica的数据模型设计最成熟(21k GitHub Stars)

建议4: 个人先用再考虑产品化

基于你的实际需求，建议先做一个"个人工具"而非"产品":

1. 先把自己的50-100个关键联系人录入并分圈

2. 跑2-3个月，积累互动数据，验证评分模型

3. 确认核心流程后再考虑产品化和商业化

4. 这样做的好处: 你就是第一个重度用户，产品方向不会偏
